# Manifesto do Parina Framework  
## *Edição Altiplano — Onde menos é mais*

O Parina não é um framework full-stack.  
Não quer ser.  
Não pretende fazer tudo.

Seu propósito é mais profundo:

> **Libertar o desenvolvedor da complexidade desnecessária.  
> Para voltar a pensar. Para voltar a ver. Para construir com intenção.**

---

# 1. O silêncio do Altiplano

Em um mundo cheio de ruído — frameworks gigantes, camadas ocultas, arquivos gerados, magia difícil de seguir —  
O Parina propõe uma alternativa:

**a simplicidade extrema como um ato de resistência.**

O Altiplano está vazio de propósito.  
E é por isso que é belo.

---

# 2. O núcleo como ferramenta, não como gaiola

O Parina oferece apenas o essencial:

- um controlador frontal (front controller)  
- um roteador explícito  
- um núcleo que despacha  
- manipuladores (handlers) independentes  
- vistas simples  
- respostas claras  

Sem modelos mágicos.  
Sem configurações ocultas.  
Sem convenções incompreensíveis.

> **Se você quiser mais, construa você mesmo.  
> É assim que você recupera a autoria do seu próprio código.**

---

# 3. A responsabilidade volta para as suas mãos

Grandes frameworks resolvem tanto que deixam você sem julgamento.  
Parina Framework não.

Aqui, o desenvolvedor:

- decide como acessar o banco de dados  
- decide como modelar seu domínio  
- decide como estruturar suas pastas  
- decide como estender  
- decide qual infraestrutura precisa  

Este framework não o infantiliza.  
Ele o respeita.

---

# 4. Tudo o que não está lá... é intencional

O Parina Framework NÃO inclui:

- ORM  
- migrações  
- filas (queues)  
- cache  
- templates complexos  
- ajudantes mágicos (magic helpers)  
- auto–scaffolding mágico/em tempo de execução (o scaffolding é explícito via CLI)  
- múltiplas camadas de transporte  

Isso é uma falha?

Não.  
É um lembrete de que **você pode criar o que realmente precisa**.

---

# 5. O código deve caber na sua cabeça

Um framework que você não entende  
é um framework que você não controla.

A arquitetura do Parina foi desenhada para caber:

- em um quadro branco,  
- em um guardanapo,  
- na sua mente.

Se algo for complexo demais para explicar em 30 segundos…  
não pertence aqui.

---

# 6. O desenvolvimento como ato criativo

O Parina Framework não entrega tudo pronto.  
Ele lhe dá a tela em branco, os pincéis e a paleta.

O artista é você.

---

# 7. A promessa final

> **"Não lhe dou um framework para programar mais rápido.  
> Dou-lhe um framework para programar melhor."**

Menos código.  
Mais clareza.  
Mais intenção.  
Mais controle.

Esse é o espírito do Parina.  
Esse é o espírito do Altiplano:  
vastidão, simplicidade, silêncio…  
e a liberdade de construir o que é essencial.

---

**Fim do manifesto.**
