# Manifeste du Framework Parina  
## *Édition Altiplano — Où moins, c'est plus*

Parina n'est pas un framework full-stack.  
Il ne veut pas l'être.  
Il ne prétend pas tout faire.

Son but est plus profond :

> **Libérer le développeur de la complexité inutile.  
> Pour penser à nouveau. Pour voir à nouveau. Pour construire avec intention.**

---

# 1. Le silence de l'Altiplano

Dans un monde plein de bruit — frameworks géants, couches cachées, fichiers générés, magie difficile à suivre —  
Parina propose une alternative :

**l'extrême simplicité comme acte de résistance.**

L'Altiplano est vide à dessein.  
Et c'est pourquoi il est beau.

---

# 2. Le noyau comme outil, pas comme cage

Parina ne fournit que l'essentiel :

- un contrôleur frontal (front controller)  
- un routeur explicite  
- un noyau qui distribue  
- des gestionnaires (handlers) indépendants  
- des vues simples  
- des réponses claires  

Pas de modèles magiques.  
Pas de configurations cachées.  
Pas de conventions incompréhensibles.

> **Si vous voulez plus, construisez-le vous-même.  
> C'est ainsi que vous récupérez la paternité de votre propre code.**

---

# 3. La responsabilité revient entre vos mains

Les grands frameworks résolvent tellement de choses qu'ils vous laissent sans jugement.  
Parina Framework non.

Ici, le développeur :

- décide comment accéder à la base de données  
- décide comment modéliser son domaine  
- décide comment structurer ses dossiers  
- décide comment étendre  
- décide de quelle infrastructure il a besoin  

Ce framework ne vous infantilise pas.  
Il vous respecte.

---

# 4. Tout ce qui n'est pas là... est intentionnel

Le Framework Parina n'inclut PAS :

- ORM  
- migrations  
- files d'attente (queues)  
- mise en cache  
- templates complexes  
- assistants magiques (magic helpers)  
- auto–scaffolding magique/au runtime (le scaffolding est explicite via CLI)  
- multiples couches de transport  

Est-ce un défaut ?

Non.  
C'est un rappel que **vous pouvez créer ce dont vous avez réellement besoin**.

---

# 5. Le code doit tenir dans votre tête

Un framework que vous ne comprenez pas  
est un framework que vous ne contrôlez pas.

L'architecture de Parina a été conçue pour tenir :

- sur un tableau blanc,  
- sur une serviette,  
- dans votre esprit.

Si quelque chose est trop complexe à expliquer en 30 secondes…  
cela n'a pas sa place ici.

---

# 6. Le développement comme acte créatif

Le Framework Parina ne vous donne pas tout prêt.  
Il vous donne la toile blanche, les pinceaux et la palette.

L'artiste, c'est vous.

---

# 7. La promesse finale

> **"Je ne vous donne pas un framework pour programmer plus vite.  
> Je vous donne un framework pour programmer mieux."**

Moins de code.  
Plus de clarté.  
Plus d'intention.  
Plus de contrôle.

C'est cela, l'esprit de Parina.  
C'est cela, l'esprit de l'Altiplano :  
l'immensité, la simplicité, le silence…  
et la liberté de construire ce qui est essentiel.

---

**Fin du manifeste.**
