# Parina Framework 宣言  
## *アルティプラーノ版 —— 最小限こそが豊かさ (Where less is more)*

Parina はフルスタックフレームワークではありません。  
それを望んでもいませんし、  
すべてを行おうともしていません。

その目的は、より深遠なものです：

> **開発者を不要な複雑さから解放すること。  
> 再び思考すること。再び見つめること。意図を持って構築すること。**

---

# 1. アルティプラーノの静寂

巨大なフレームワーク、隠されたレイヤー、自動生成されるファイル、追いきれない魔法 —— ノイズに満ちた世界において、  
Parina はひとつの選択肢を提案します：

**極限のシンプルさを、ひとつの抵抗行為とすること。**

アルティプラーノ（高地平原）が空っぽなのは、意図的なものです。  
だからこそ、美しいのです。

---

# 2. ツールとしてのコア、檻ではない

Parina は本質的なものだけを提供します：

- フロントコントローラー (front controller)  
- 明示的なルーター (explicit router)  
- 処理を振り分けるカーネル (kernel)  
- 独立したハンドラー (handlers)  
- シンプルなビュー (views)  
- 明快なレスポンス (responses)  

魔法のようなモデルはありません。  
隠された設定はありません。  
理解しがたい規約はありません。

> **それ以上を望むなら、自分で構築してください。  
> そうしてこそ、自らのコードの主体性を取り戻すことができるのです。**

---

# 3. 責任は再びあなたの手に

巨大なフレームワークはあまりにも多くのことを解決するため、あなたから判断力を奪ってしまいます。  
Parina Framework は違います。

ここでは、開発者が以下のことを決定します：

- データベースへのアクセス方法  
- ドメインのモデリング方法  
- フォルダの構造化方法  
- 拡張の方法  
- 必要なインフラ  

このフレームワークは、あなたを子供扱いしません。  
あなたを尊重します。

---

# 4. そこにないものすべて……それは意図されたものです

Parina Framework には以下が含まれていません：

- ORM  
- マイグレーション  
- キュー (queues)  
- キャッシング  
- 複雑なテンプレートエンジン  
- 魔法のヘルパー (magic helpers)  
- 魔法の/実行時自動スカフォールディング (スカフォールディングは CLI を通じて明示的に行われます)  
- 複数のトランスポートレイヤー  

それは欠点でしょうか？

いいえ。  
それは、**あなたが本当に必要とするものを自ら作り出せる**という気づきなのです。

---

# 5. コードはあなたの頭に収まらなければならない

理解できないフレームワークは、  
制御できないフレームワークです。

Parina のアーキテクチャは、以下に収まるように設計されています：

- ホワイトボードの上に、  
- 紙ナプキンの上に、  
- あなたの頭の中に。

もし何かが 30 秒で説明できないほど复杂であるなら……  
それはここに属すべきではありません。

---

# 6. 創造的行為としての開発

Parina Framework はすべてを用意してはくれません。  
真っ白なキャンバス、絵筆、 painted palette を与えてくれます。

芸術家はあなた自身です。

---

# 7. 最後の約束

> **「コードを速く書くためのフレームワークを与えるのではありません。  
> コードをより良く書くためのフレームワークを与えるのです」**

より少ないコード。  
より多くの明快さ。  
より多くの意図。  
より多くの制御。

それが Parina の精神です。  
それがアルティプラーノの精神です：  
広大さ、シンプルさ、静寂……  
そして本質的なものを構築する自由。

---

**宣言は以上です。**
